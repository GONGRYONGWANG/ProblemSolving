### 문제 설명

<p>0점에서 시작해 매 턴 점수를 획득해 나가는 게임이 있다. 각 턴에서 플레이어가 게임 에서 승리하면 <em>a</em>점을 얻고, 지면 <em>b</em>점을 얻는다. 점수의 총합이<em> n</em>점 이상이면 게임이 끝나게 된다.</p>

<p>포스텍의 한 학생이 이 게임을 해킹하는 데 성공했다. 그래서 각 턴에서 승리할지, 질지 자유롭게 결정할 수 있다. 심지어는 원하는 턴에서 점수를 <em>a</em>점이나 <em>b</em>점 올리는 대신 두 배로 바꾸는 것도 가능하다. 예를 들면 <em>a</em>가 3이고 <em>b</em>가 2이며 현재 점수가 10이라고 할 때, 다음 턴의 점수로 가능한 것에는 12, 13, 또는 20이 있다. 하지만 만약 학생이 <em>n+a</em>점 이상의 점수로 게임을 끝내게 되거나, 각 턴이 끝났을 때 점수를 두 배로 올린 턴이 전체 턴의 10%를 초과하면 학생은 게임 운영진의 의심을 받게 되고, 게임에서 퇴출당하게 된다.</p>

<p>이 학생이 운영진의 의심을 받지 않고 게임을 끝내는 데 필요한 최소한의 턴을 구하라.</p>

### 입력 

 <p><em>n, a, b</em> 세 개의 정수가 띄어쓰기를 사이에 두고 주어진다. (1 ≤ <em>n </em>≤ 500, 1 ≤ <em>b</em> ≤ <em>a</em> ≤ 100)</p>

### 출력 

 <p>이 학생이 운영진의 의심을 받지 않고 게임을 끝내는 데 필요한 최소한의 턴을 출력한다.</p>
